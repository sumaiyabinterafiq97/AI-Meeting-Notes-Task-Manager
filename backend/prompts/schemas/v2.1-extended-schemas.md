# MeetingMind AI — v2.1 Extended Output Schemas (Draft for Review)

**Status:** Approved — prompt templates v2.1.0 shipped  
**Prompt version:** 2.1.0  
**Feature flag:** `PROMPT_SCHEMA_V2_1=true` (backend schema validation until `agent-schemas.ts` migrated)  
**Baseline:** v2.0 strict schemas in `response-format.prompt.md` / `agent-schemas.ts`

---

## 1. Review summary

v2.1 adds **structured confidence, ownership, and evidence fields** without breaking v2.0 merge compatibility. New fields are additive; merge layer strips or maps extensions until DB columns exist.

| Agent | New fields | Merge impact | DB migration |
|-------|------------|--------------|--------------|
| Meeting Summary | `nextSteps`, `participantsDiscussed` | Map `keyTopics` → `topics` (unchanged) | Optional JSON column |
| Task Extractor | `priority`, `dependencies`, `confidenceScore` | Strip before `action_item_suggestions` until migrated | Extend suggestions table |
| Decision | `stakeholders`, `confidenceScore`, `supportingEvidence` | Strip; `context` remains canonical | Optional JSON on decisions |
| Risk Analyzer | `impact`, `likelihood`, `recommendation`, `evidence`, `confidenceScore` | Strip; `severity` + `context` remain | Optional JSON on risks |
| Weekly Report | `citations[]` per section | Additive in `sections` | None if optional |
| Knowledge | `sourceRef` | Additive | `knowledge_entries.source_ref` |
| Chat | Structured `citations` in done event | Already parsed | None |

**Recommendation:** Ship v2.1 prompts behind flag; validate with YAML fixtures; migrate DB in Phase 2.

## 2. Meeting Summary — `MeetingSummaryOutputV21`

### JSON Schema

```json
{
  "$id": "MeetingSummaryOutputV21",
  "type": "object",
  "properties": {
    "summary": { "type": "string", "maxLength": 4000 },
    "keyTopics": {
      "type": "array",
      "items": { "type": "string", "maxLength": 120 },
      "minItems": 0,
      "maxItems": 8
    },
    "nextSteps": {
      "type": "array",
      "items": { "type": "string", "maxLength": 300 },
      "maxItems": 5,
      "description": "Logistical follow-ups only — NOT action items"
    },
    "participantsDiscussed": {
      "type": "array",
      "items": { "type": "string" },
      "description": "Speakers who contributed materially"
    }
  },
  "required": ["summary", "keyTopics"],
  "additionalProperties": false
}
```

### Merge mapping (v1 compatibility)

```json
{
  "summary": "<from summary>",
  "topics": "<from keyTopics>",
  "nextSteps": "stored in meeting_ai_outputs.metadata.nextSteps (v2.1)"
}
```

### Review decisions (approved 2026-06-18)

- [x] `nextSteps` stored in metadata; not merged into summary text (legacy UI unchanged)
- [x] Max 5 `nextSteps` sufficient
- [x] Task items with `confidenceScore` < 0.7 omitted from output
- [x] Decision `confidenceScore` < 0.60 omitted
- [x] `supportingEvidence` max 500 chars required when decision extracted

## 3. Task Extraction — `TaskExtractorOutputV21`

### JSON Schema

```json
{
  "$id": "TaskExtractorOutputV21",
  "type": "object",
  "properties": {
    "actionItems": {
      "type": "array",
      "maxItems": 20,
      "items": {
        "type": "object",
        "properties": {
          "title": { "type": "string", "maxLength": 300 },
          "description": { "type": "string", "maxLength": 1000 },
          "suggestedAssignee": { "type": ["string", "null"] },
          "suggestedDueDate": {
            "type": ["string", "null"],
            "pattern": "^\\d{4}-\\d{2}-\\d{2}$"
          },
          "priority": {
            "type": ["string", "null"],
            "enum": ["low", "medium", "high", "critical", null]
          },
          "dependencies": {
            "type": "array",
            "items": { "type": "string", "maxLength": 200 },
            "maxItems": 5
          },
          "confidenceScore": {
            "type": "number",
            "minimum": 0,
            "maximum": 1,
            "description": "≥0.7 recommended for auto-suggest surfacing"
          }
        },
        "required": [
          "title",
          "description",
          "suggestedAssignee",
          "suggestedDueDate",
          "priority",
          "dependencies",
          "confidenceScore"
        ],
        "additionalProperties": false
      }
    }
  },
  "required": ["actionItems"],
  "additionalProperties": false
}
```

### Field semantics

| Field | Rules |
|-------|-------|
| `priority` | `critical` only when explicit urgency ("P0", "blocker", "ASAP today") |
| `dependencies` | Other task titles or external blockers referenced in transcript |
| `confidenceScore` | 0.9+ explicit assignment; 0.7–0.9 implied; <0.7 omit item |

### Merge mapping

v2.0 fields persist to `action_item_suggestions`. v2.1 fields → `metadata` JSONB until columns added.

## 4. Decision — `DecisionOutputV21`

### JSON Schema

```json
{
  "$id": "DecisionOutputV21",
  "type": "object",
  "properties": {
    "decisions": {
      "type": "array",
      "maxItems": 15,
      "items": {
        "type": "object",
        "properties": {
          "text": { "type": "string", "maxLength": 500 },
          "context": { "type": "string", "maxLength": 1000 },
          "stakeholders": {
            "type": "array",
            "items": { "type": "string" },
            "maxItems": 10,
            "description": "Names from memberNames involved or affected"
          },
          "confidenceScore": { "type": "number", "minimum": 0, "maximum": 1 },
          "supportingEvidence": {
            "type": "string",
            "maxLength": 500,
            "description": "Verbatim or near-verbatim transcript excerpt"
          }
        },
        "required": ["text", "context", "stakeholders", "confidenceScore", "supportingEvidence"],
        "additionalProperties": false
      }
    }
  },
  "required": ["decisions"],
  "additionalProperties": false
}
```

### Confidence rubric

| Score | Meaning |
|-------|---------|
| 0.95+ | Explicit "we decided/agreed/approved" |
| 0.80–0.94 | Strong consensus, informal closure |
| 0.60–0.79 | Implicit agreement — **omit unless policy allows** |
| <0.60 | Do not extract |

## 5. Risk Analyzer — `RiskAnalyzerOutputV21`

### JSON Schema

```json
{
  "$id": "RiskAnalyzerOutputV21",
  "type": "object",
  "properties": {
    "risks": {
      "type": "array",
      "maxItems": 10,
      "items": {
        "type": "object",
        "properties": {
          "text": { "type": "string", "maxLength": 500 },
          "severity": { "enum": ["low", "medium", "high"] },
          "context": { "type": "string", "maxLength": 1000 },
          "impact": { "type": "string", "maxLength": 500 },
          "likelihood": { "enum": ["low", "medium", "high", "unknown"] },
          "recommendation": { "type": "string", "maxLength": 500 },
          "evidence": { "type": "string", "maxLength": 500 },
          "confidenceScore": { "type": "number", "minimum": 0, "maximum": 1 }
        },
        "required": [
          "text",
          "severity",
          "context",
          "impact",
          "likelihood",
          "recommendation",
          "evidence",
          "confidenceScore"
        ],
        "additionalProperties": false
      }
    }
  },
  "required": ["risks"],
  "additionalProperties": false
}
```

### Severity × likelihood matrix (guidance)

| likelihood \\ severity | low | medium | high |
|------------------------|-----|--------|------|
| high | medium | high | high |
| medium | low | medium | high |
| low | low | low | medium |
| unknown | use severity only | | |

`recommendation`: only if stated in transcript; else `"Monitor"` or empty string.

## 10. Regression risks

| Change | Risk | Mitigation |
|--------|------|------------|
| Required `confidenceScore` | More empty arrays if model conservative | Calibrate few-shots; threshold 0.6 for tasks |
| `supportingEvidence` required | Token bloat | Max 500 chars |
| Strict enum on `likelihood` | Validation failures | Allow `unknown` |
| Task `priority` required | Null vs omitted in strict mode | Use `["string","null"]` enum pattern |
